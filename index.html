    Firefox OS


    Também conhecido como B2G (Boot to Gecko), o Firefox OS é uma plataforma para dispositívos móveis.


    Seu alicerce é constituído pelo HTML5 e pelas Web APIs.


    Liberdade de plataformas proprietárias. Escreva uma vez, execute onde quiser!

    Gaia


    O Gaia é a interface gráfica com o usuário no Firefox OS.


    Tudo que você precisa saber para criar aplicativos são tecnologias web como JavaScript, HTML e CSS.


    As APIs de acesso são definidas pela W3C DAP (DeviceAPis).

    Gecko


    Motor de renderização que está presente em diversos navegadores {Firefox, SeaMonkey, Camino etc}


    Camada de execução de serviços. Interação entre o Gaia e as APIs.


    Gonk


    Sistema Operacional low-level, Kernel Linux e Camada de abstração de Hardware


    Kernel baseado no AOSP (Android Open Source Project)


    É uma camada de portabilidade. Isso é, fornece uma interface entre hardware e o Gecko.

    • Media Query


    • Media Queries foram criadas para tornar o HTML portável.
    • Pluralidade de plataformas dificulta a criação de um css único.
    • Através da análise de condições, estilos são aplicados.
      • width, height, orientation, aspect-ratio, color, resolution etc..
    • Forte aliado no desenvolvimento para plataformas mobile, pois cada aparelho possui características próprias.

    Media Query



                            /* regra aplicada em todo lugar */
                        body { background: blue; }
                         
                        /* aplica somente a partir de 320px */
                        @media screen and (min-width: 320px) {
                           body { font-size: 80%; }
                        }
                         
                        /* aplica somente a partir de 480px em landscape */
                        @media screen and (min-width: 480px) and (orientation: landscape) {
                           nav { float: left; }
                        }

    Web APIs

    O que são?


    • As WebAPIs oferecem acesso direto ao hardware através de JavaScript (sim, isso é magico!)
    • As WebAPIs são padronizadas e desenvolvidas em conjunto com a W3C. Com isso, seu app precisará cada vez menos de alterações para plataformas diferentes.
    • Estão sendo implementadas no Firefox desktop e mobile. Desta forma, seu app é independente da plataforma.

    Quais são as funcionalidades?

    • Informação de Rede, Bluetooth, Conexão Móvel(3G), Estatísticas de Rede, TCP Socket, Telefonia.
    • WebSMS, Informações do WiFi, Sensor de Luz Ambiente, Estado da Bateria, Geolocalização, Bloqueio de Mouse (Pointer Lock)
    • Proximidade, Orientação, Vibração, WebFM, Camera, Gerenciamento de energia
    • Manipulação de Arquivos, IndexedDB, Contatos, Armazenamento, Configurações
    • Alarme, Notificações, PUSH, Web Activities (Delegação de tarefas), WebPayment, Browser, Permissões, Tempo/Relógio

    Tipos de Aplicativo


      Hospedado

    • Ficam armazenados em um servidor externo. É de obrigação do usuário fornecer a hospedagem e manutenção do up-time do app.
    • Em caso do app não estar disponível no cache do dispositivo, o servidor remoto será consultado.
    • Vantagens
      • Facilidade de Deploy e utilização do Github como servidor
    • Desvantagens
      • Utilização do appcache

    Tipos de Aplicativo


      Empacotado

    • São aplicativos comprimidos no formato zip, onde todos os dados necessários estão inclusos.
    • Para atualizar o aplicativo, é necessário seguir o processo definido pelo Firefox Marketplace.
    • Vantagens
      • Velocidade de execução, agilidade no desenvolvimento.
    • Desvantagens
      • Processo de distribuição do Marketplace

    Níveis de acesso ao hardware


      Normal

    • Aplicativos hospedados e aplicativos empacotados que não declaram um tipo no manifesto são atribuídos a esse grupo.

    • Acesso a APIs como geolocalização, PUSH, Armazenamento, captura de vídeo, áudio, rádio FM entre outras.

    Níveis de acesso ao hardware


      Privilegiado


    • A principal diferença entre o nível normal e o privilegiado, é que para obter o status de privilegiado ele deve ser empacotado, ao contrário da normal onde apps podem tanto ser empacotados ou hospedados.

    • Acesso a APIs como Socket TCP, Informações sobre a Rede, Armazenamento e Leitura do cartão SD, manipulação de arquivos de vídeo e áudio entre outras.

    Níveis de acesso ao hardware


      Certificado


    • Aplicativos certificados tem acesso total ao hardware do dispositivo. Infelizmente, aplicativos certificados ou são da Mozilla ou OEM (parceiros de hardware).

    • Acesso as APIs críticas e de controle do aparelho. Todos os aplicativos nativos são do tipo certificado.

    Arquivo Manifesto


    • O manifesto é um arquivo do tipo JSON que descreve o aplicativo.
    • Deve ser nomeado manifest.webapp e ficar na raiz do projeto.
    • Existem campos obrigatórios e campos que variam de acordo com as necessidades do seu aplicativo.
    • Quando um aplicativo é instalado de forma remota, o arquivo manifest é responsável por fornecer as informações necessárias.

    Firefox Marketplace

    O que é?


    • O Marketplace é uma loja de aplicativos projetados para os aparelhos Firefox OS.
    • Aplicativos para o Firefox Desktop e Firefox para Android também estão disponíveis.
    • O Marketplace disponibiliza aplicativos gratuitos e pagos. Você pode utilizar o Persona para compartilhar aplicativos entre aparelhos.
    • Muitos aplicativos já foram portados para o Firefox OS, e o número só cresce.

    Vantagens


    • API aberta para executar pagamentos in-app
    • Monetização através de anúncios em aplicativos
    • Oportunidade de criar apps de qualidade
    • Baixa curva de aprendizado e processo de disponibilização de app facilitado.
